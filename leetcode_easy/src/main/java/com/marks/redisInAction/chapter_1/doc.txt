使用java 和 JRedis 实现后端代码接口, 前端页面无需实现

对文章进行投票
文章的数据结构
{
article: 92617 (文章的id)
title: Go to statement considered harmful (文章标题)
link: http://goo.gl/kZUSu (文章链接)
poster: [user: 83271 (user id)] (发布者)
time: 1331382699.33 (发布时间, Unix 时间)
votes: 528 (获取的投票数量)
}

规则如下:
1. 如果一篇文章获取至少200张支持票, 那么可以认为这篇文章是一篇有趣的文章.
2. 假设网址每天发布1000篇文章, 而其中至少有50篇文章是有趣的文章。那么网站需要将这 50 篇文章
放在文章列表前 100 位至少1天.
3. 为了使得评分随着时间而降低, 程序会根据当前时间和文章发布时间来计算当前文章的评分, 具体计算方法如下:
int score = time + votes * 432; 其中 432 为常量,
即认定一篇有趣文章需要1天内获取 200 张支持票, 86400/200 = 432, 即每张支持票提供 432的得分.
4. 文章的排序有两种规则, 一种是根据发布时间排序, 一种是根据分数高低排序
5. 防止同一用户多次对一篇文章进行投票, 需要记录每一篇文章的投票人id. 并且为了节约内存, 文章的投票有效时间为一周,
一周后该文章无法进行投票, 并且删除该篇文章的投票人id 信息.

代码需要如下功能:
1. 文章投票接口 articleVote, 参数是: article id, user id;
需要判断当前文章是否超过投票有效期, 如果没有超过投票有效期, 判断是否是重复投票(即同一个 user id), 如果都不是, 则增加当前文章的投票数, 更新文章的分数值.

2. 发布文章接口 postArticle, 参数是: user id(即poster id), title, link
需要通过 redis创建一个新的文章 id, 通过计数器执行 INCR 命令完成. 然后将 user id 添加到当前文章已投票的集合中, 设置当前文章创建时间和投票有效时间(1周)。
添加当前文章到两个排序的集合中(时间和分数).

3. 获取文章排序后的信息(可以添加分页信息), 参数: page, 排序方式(time/score)
设定网站中页面中一页显示25篇文章信息, 当前接口返回文章信息(article id, title, link, poster, time, votes)

4. 由于每一篇文章都有一个额外的分组信息, 需要对文章进行分组操作, 添加接口分组
