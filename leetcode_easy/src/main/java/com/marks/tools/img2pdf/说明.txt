1. 找到一个问题, 即由于 ImageToPdfConverterAdvanced(java 代码) 创建最大的单页, 14400 * 14400, 这是导致内存占用过高并且读取文件过慢的原因.
2. 能有效降低内存消耗, 减少无用的方法是, 将单页分割成10页.
3. 即当前是使用 100 张图片, 来拼接成一个最大的单页 14400 * 14400, 这会造成空间的巨大浪费, 图片的内容只占据中间的 14400 * 1000,
现在需要将10张图片拼接成pdf中的页面, 该页的大小设置为 3000 * 3000, 总的消耗是 3000 * 3000 * 10.
4. 如果需要实现多页, 并且需要让整个pdf文件呈现紧凑的布局
5. 单页中图片的间隔设置为 1px, 另外页面与页面之间的间隔设置为2px
6. 由于单页限制大小为 3000 * 3000, 大部分情况下10张图片的长宽会超过页面限制大小, 需要整体对图片进行缩放操作.
