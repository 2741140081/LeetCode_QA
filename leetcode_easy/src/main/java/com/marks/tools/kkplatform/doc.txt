完整游戏流程如下:
1. 切换到准备房间界面(后续会给出app 名称以及相关信息, 确保可以正确切换的改页面上)
2. 通过图片识别(最佳缩放和迭代二分查找最佳), 找到按钮"开始游戏", 点击按钮, 游戏加载(1min 左右), 加载完成后, 游戏界面提示"按任意键继续"
3. 切换到游戏界面(后续会给出app 名称以及相关信息, 确保可以正确切换的改页面上), 按下"空格键"(任意键, 习惯性用空格, 其他键也可), 进入游戏
4. 通过图片识别, 找到对应的难度选项, 点击选择难度
5. 选择英雄, 英雄通过(F1, F2, F3) 的按钮进行选择, 自动化过程中只选择F1 的英雄, 按下F1 键, 通过图片识别, 找到"确认"按钮, 点击确认,
按下组合键 "Ctrl + 1", 对英雄进行编号(当按下 1键时, 会选择该英雄)
6. 切换到修改器app 界面, 通过图片识别, 找到并且点击"查找游戏"按钮
7. 通过图片识别, 在树形结构中找到并且点击"人物属性" 选项
8. 通过图片识别, 在右侧具体属性栏中, 通过图片识别找到"经验值" 输入框(这个可以识别, 标记, 并且保存截图), 双击输入框 2/3 位置处,
Ctrl + A 全选内容, 输入 "45000000" , Enter 键确认
9. 通过图片识别, 找到并点击 "修改"按钮
10. 切换到游戏界面, 找到并且点击天赋按钮(一个图标), 图片识别并且点击 "火焰天赋" 按钮
11. 按下 "1键" 选择英雄, 找到并点击 "准备" 按钮
12. 鼠标一直位于 "准备" 按钮位置处, 每隔 5s 点击一次"准备"按钮
13. 过了15分钟后, 识别"存档"按钮, 是否存在, 如果不存在, 延迟30s后, 再次识别"存档" 按钮,
如果存在, 点击"存档" 按钮, 找到并且点击 "开始存档" 按钮, 图片识别 并且 点击 "存档属性A" 选项
14. 在13步骤完成后, 图片识别 并点击 "退出游戏" 按钮

1~14 步骤是一个完整游戏流程, 重复执行该流程即可

KingOfBeastsArchive: 游戏界面的相关操作方法
ModifiersOperation: 修改器界面操作相关方法
common/GameOperationCommon: 公共类, 点击按钮操作, 查找图片操作, 这些操作方法可以抽象为一个方法放在公共类中, 供其他类进行调用

solution:
1. 设计存档信息表, 数据库使用mysql数据库
表名: 存档装备强化信息(archive_equipment_enhancement_information), 简称 AEE_Info
表字段: 自增字段 AE_id 类型 int, 存档装备编号 AE_Number 类型 int, 存档装备名称 AE_Name 类型 String, 存档装备图片名称 AE_Img_Name 类型 String,
存档装备图片路径 AE_Img_Path 类型 String, 存档装备所在页面 AE_Page_Number 类型 int, 存档装备已强化次数 AE_Enhanced_Count 类型 int,
存档装备最大强化次数 AE_Max_Enhancement_Count 类型 int, 更新日期 Update_Time 类型 timestamp, 存档装备描述 AE_Description 类型 String

表创建的sql:
CREATE TABLE archive_equipment_enhancement_information (
    AE_id INT NOT NULL AUTO_INCREMENT COMMENT '自增主键ID',
    AE_Number INT NOT NULL COMMENT '存档装备编号',
    AE_Name VARCHAR(255) NOT NULL COMMENT '存档装备名称',
    AE_Img_Name VARCHAR(255) COMMENT '存档装备图片名称',
    AE_Img_Path VARCHAR(500) COMMENT '存档装备图片路径',
    AE_Page_Number INT NOT NULL DEFAULT 1 COMMENT '存档装备所在页面',
    AE_Enhanced_Count INT NOT NULL DEFAULT 0 COMMENT '存档装备已强化次数',
    AE_Max_Enhancement_Count INT NOT NULL DEFAULT 50 COMMENT '存档装备最大强化次数',
    Update_Time TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新日期',
    AE_Description TEXT COMMENT '存档装备描述',
    PRIMARY KEY (AE_id),
    INDEX idx_ae_number (AE_Number),
    INDEX idx_update_time (Update_Time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='存档装备强化信息表';
