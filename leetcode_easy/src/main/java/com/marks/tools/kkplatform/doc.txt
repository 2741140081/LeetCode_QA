大部分问题已经解决了, 后续可能遇到问题时, 具体问题具体分析, 现在需要把大致的代码框架搭建起来:
1. 目标: 构建这个代码主要是为了自动化, 在进行游戏时, 自动化通过游戏并且获得存档
2. 游戏名称: King of Beasts(斗兽之王), 每局游戏时间大概是 20分钟左右
3. 游戏修改器app: xTools.exe 软件, 可以修改当局游戏的部分数据, 不需要关心具体实现, 只需要关注修改器的按钮及其相关输入框之类
4. 游戏过程中, 只需要关注于图片识别和点击事件即可
5. 难点问题之前的代码已经解决, 主要是应用切换和查找图片, 以及图片的最佳缩放比例

完整游戏流程如下:
1. 切换到准备房间界面(后续会给出app 名称以及相关信息, 确保可以正确切换的改页面上)
2. 通过图片识别(最佳缩放和迭代二分查找最佳), 找到按钮"开始游戏", 点击按钮, 游戏加载(1min 左右), 加载完成后, 游戏界面提示"按任意键继续"
3. 切换到游戏界面(后续会给出app 名称以及相关信息, 确保可以正确切换的改页面上), 按下"空格键"(任意键, 习惯性用空格, 其他键也可), 进入游戏
4. 通过图片识别, 找到对应的难度选项, 点击选择难度
5. 选择英雄, 英雄通过(F1, F2, F3) 的按钮进行选择, 自动化过程中只选择F1 的英雄, 按下F1 键, 通过图片识别, 找到"确认"按钮, 点击确认,
按下组合键 "Ctrl + 1", 对英雄进行编号(当按下 1键时, 会选择该英雄)
6. 切换到修改器app 界面, 通过图片识别, 找到并且点击"查找游戏"按钮
7. 通过图片识别, 在树形结构中找到并且点击"人物属性" 选项
8. 通过图片识别, 在右侧具体属性栏中, 通过图片识别找到"经验值" 输入框(这个可以识别, 标记, 并且保存截图), 双击输入框 2/3 位置处,
Ctrl + A 全选内容, 输入 "45000000" , Enter 键确认
9. 通过图片识别, 找到并点击 "修改"按钮
10. 切换到游戏界面, 找到并且点击天赋按钮(一个图标), 图片识别并且点击 "火焰天赋" 按钮
11. 按下 "1键" 选择英雄, 找到并点击 "准备" 按钮
12. 鼠标一直位于 "准备" 按钮位置处, 每隔 5s 点击一次"准备"按钮
13. 过了15分钟后, 识别"存档"按钮, 是否存在, 如果不存在, 延迟30s后, 再次识别"存档" 按钮,
如果存在, 点击"存档" 按钮, 找到并且点击 "开始存档" 按钮, 图片识别 并且 点击 "存档属性A" 选项
14. 在13步骤完成后, 图片识别 并点击 "退出游戏" 按钮

1~14 步骤是一个完整游戏流程, 重复执行该流程即可

GameStarter: 整个自动化流程的启动类, 包含了切换界面功能
PrepareRoom: 准备房间界面功能, 主要是点击 "开始游戏"
KingOfBeastsArchive: 游戏界面的相关操作方法
ModifiersOperation: 修改器界面操作相关方法
common/GameOperationCommon: 公共类, 点击按钮操作, 查找图片操作, 这些操作方法可以抽象为一个方法放在公共类中, 供其他类进行调用
