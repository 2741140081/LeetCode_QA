1. 目标网站: https://www.target.com (漫画url, 待定)
2. 该网站有漫画(目录 + 内容)
3. 首先需要提取目标漫画的所有目录 id, 拼接为 目录(章节) url
4. 每一个章节的url页面中, 保存了该章节的所有的漫画图片, 大概率通过id(待定) 进行提取相应信息
5. 对于每一个章节, 使用多线程进行提取每个章节下的所有漫画内容(图片url信息)
6. 每一个章节包含图片url(50 ~ 100) 条url信息, 然后在通过多线程根据图片url 进行图片下载
7. 并且每一个章节下的图片信息都会有一个唯一的 "img + num".jpg, 例如 img0.jpg 这个的唯一标识的图片id。
也就是说, 每一个章节下图片 id 是从 img0 ~ img100 之间的一个id 数,
8. 将同一个章节下的图片进行竖向合并, 保存为pdf文件, pdf 文件名称为漫画的章节名称, pdf 个数为 目标漫画的章节数, 保存在一个目录(待定)下
9. 清理磁盘, 清理目标漫画下载的图片, 仅保留pdf文件, 清除图片, 释放磁盘空间
10. 可供参考的目录
    a. img2pdf: 将图片转为pdf代码;
    b. spider: 爬取目标网站图片的代码, 需要注意的是网站不同, spider 包下爬取的网站每一个章节分为多个url, 而当前目标网站漫画每一个章节只有一个url(更简单);
    c. download_txt_multi_threading: 多线程爬取小说内容网站代码, 可以参考结构, 即都是一个章节对于一个页面, 目标漫画网站目录可能没有分页(待定)
11. 尽量减少使用内部类, 而是将这些实体类存放到 /download_cartoon/entity 包下面, 工具类和常量类放在 /download_cartoon/utils 包下

额外处理
1. 如果图片下载失败了, 那么整个章节的pdf 就需要等待图片再次下载后重新生成, 可以写一个方法, 从新生成下载图片失败章节的pdf,
后续图片下载可能手动下载或者通过其他方式进行下载, 然后这个失败章节的图片应该进行保留, 以便手动下载失败图片后可以放在该章节的目录下, 重新生成pdf
2. 需要在CartoonDownloader 程序中输出下载失败的图片信息, 方便进行手动下载或者其他方式进行下载, 需要信息(图片id, 图片url, 图片章节id).
另外给出下载失败章节信息
