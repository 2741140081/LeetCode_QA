《小偷偷金TD》 游戏流程说明
1. 与 kkplatform 包下的 PrepareRoom 相似, 同样是准备房间, 然后点击开始游戏
2. 选择难度, 分为 初级, 中级, 高级难度三种, 其中初级难度范围是: [难度一~难度十], 中级难度是: [难度十一 ~ 难度二十], 高级难度是: [难度二十一 ~ 难度三十],
当前主要测试是中级难度中的难度十六.
3. 选择模式, 有4种模式, 选择模式四即可
4. 选择挑战, 挑战有5种类型, 可以多选, 即挑战1 和 挑战2 可以同时被选择, 当前多选挑战1~挑战4, 不选挑战5
5. 挑战选择完成后, 点击开始游戏, 正式开始游戏
6. 游戏中主要人物以及建筑物:
a. 小偷英雄, 需要用'1' 进行编号, 方便选取. 行为: 通过'A' 键指向金矿获取金币; 物品栏存放商店购买的武器, 这些武器需要给储物柜进行修改(通过修改器修改)
b. boss挑战建筑: 编号为'3', 用来开启boss 挑战用, 以及挑战最终boss
c. 属性强化建筑: 编号为'4', 通过使用金币来强化小偷的个性属性
d. 金矿建筑: 主要被小偷'A'键攻击后, 小偷能获取金币
e. 储物柜建筑: 编号为'2', 拾取小偷丢弃的武器物品, 通过修改器修改. 修改完成后, 将修改后的物品移交给小偷
f. 商店建筑: 编号5, 购买武器处, 需要选择最左上角处购买最便宜的武器
g. 各类存档挑战建筑: 游戏胜利后, 需要选取存档挑战建筑, 点击挑战存档boss
7. 整体游戏流程分为: 15分钟游戏主体 + 2 分钟的最终boss 挑战 + 游戏胜利后 3 分钟左右的存档boss 挑战内容, 预计每局用时 20 分钟


设计整体游戏脚本的流程
1. 给人物和待编号的建筑物进行编号操作, 通过组合键 'Ctrl + number(1,2,3,4)'
2. 编号3, boss挑战处, 点击开启自动挑战金币怪
3. 编号4, 属性强化处, 点击2下强化攻击速度, 间隔 1s
4. 编号5, 需要等待金币足够后, 购买武器, 等待的时间暂定是1分钟
5. 当小偷购买武器后, 需要将武器交给储物柜进行修改武器操作: 点击小偷技能栏的丢弃物品按钮, 移动鼠标到指定丢弃地点, 点击左键丢弃物品
6. 当小偷丢弃物品后, 储物柜需要将丢弃的物品进行拾取操作: 通过编号2切换到储物柜, 移动鼠标到丢弃地点,
判断是否有绿色宝箱存在, 如果存在, 左键点击绿色宝箱进行拾取操作. 如果不存在
判断是否有红色宝箱存在, 如果存在, 左键点击红色宝箱进行拾取操作. 如果不存在, 截图保存并且报错终止程序
7. 当储物柜拾取物品后, 切换窗口到修改器界面, 点击'查找游戏' 按钮, 找到'物品信息' 栏, 返回坐标点, 修改坐标点, 使得坐标中心点位于第一件物品处,
点击物品名称后, 使用组合键 'Ctrl + A' 全选物品名称, type 所需物品名称 如 'I291'(只是一个例子, 需要实际情况修改), 点击'修改' 按钮, 返回游戏窗口
8. 修改物品后, 储物柜需要将修改后的物品放置到丢弃地点, 操作为: 通过图片匹配物品栏, 鼠标移动到物品坐标点, 右键点击(按下并且释放), 移动鼠标到丢弃地点,
点击左键丢弃物品
9. 修改后的物品丢弃在丢弃地点后, 通过编号1, 切换的小偷, 左键点击小偷技能栏的'范围拾取' 按钮, 鼠标移动到丢弃地点, 再次进行左键点击操作, 即成功拾取修改后的物品
10. 小偷需要重新使用'A' 键攻击金矿获取金币.
11. 等待小偷获取 1000 金币(2分钟左右, 看具体情况修改), 通过编号2切换到储物柜
12. 点击储物柜的技能按钮, 购买5次'晕锤'技能书, 这个技能书会存放在储物柜的物品栏处.
13. 切换到修改器界面, 点击查找游戏, 查找 '物品信息栏', 通过计算得到 5 个物品的位置, 都是表格的形式, 单元格的大小是固定的, 可以通过计算得到5个物品的具体坐标信息,
与之前修改物品类似, Ctrl + A, 然后type 'I290', 'I293' 之类, 每修改一件物品, 点击一次修改操作, 5件物品修改完成后, 切换到游戏窗口
14. 储物柜将物品栏的5件物品通过右键点击的方式, 丢弃到丢弃地点, 一共需要进行5次, 每次丢弃一件物品
15. 切换到编号1, 小偷点击范围拾取, 只需要进行一次, 可以拾取5件物品
16. 切换到编号4, 每隔1分钟, 左键点击各项强化属性一次, 顺序点击(具体待定)
17. 检查游戏当前运行时间 - 游戏正式开始时间(游戏流程说明中的步骤5, 点击开始游戏), 判断是否游戏15分钟了, 如果没有则继续执行步骤16并等待
18. 到达15分钟后, 编号切换到3, 点击开启最终 boss 挑战, 等待1分钟后
19. 每隔10s 判断'游戏胜利' 按钮是否存在, 如果存在, 则证明boss 以及被 kill 了, 可以进行存档boss 挑战操作
20. (圈选/点击) 存档 boss 挑战建筑, 点击存档boss 挑战, 共有4处存档boss 挑战建筑, 每个建筑内的挑战boss 数量不一致, 依次点击
21. 等待3分钟, 等小偷 kill 所有存档boss, exitGame() 方法执行, 整个游戏流程结束


图片存储路径: D:/images/automation/thief_gold/ 文件夹下, 子目录存储不同的图片
/common 文件夹: 存放各种建筑图片, 例如储物柜, boss 挑战建筑等; 绿色和蓝色宝箱图; 丢弃地点图片; '游戏胜利' 按钮图片;
/start 文件夹: 存储各种难度图片, 模式图片, 挑战图片
/1 小偷: 存储小偷的各种技能图片, 例如丢弃物品技能图片, 和 范围拾取技能图
/2 储物柜: 存储武器种类图片(大致有3种物品图片); 储物柜技能栏图片('晕锤' 图片);
/3 boss挑战处: 自动挑战金币怪图片, 开启最终boss 的图片;
/4 属性强化处: 攻速强化图片, 射程强化图片, 多重射强化图片, 攻击1/2/3/4强化图片(有4种攻击强化)
/5 商店处: 购买宝箱图片
/archiver_1 存档挑战建筑1: 1存档挑战建筑中各种挑战boss图标
/archiver_2 存档挑战建筑2: 2存档挑战建筑中各种挑战boss图标
/archiver_3 存档挑战建筑3: 3存档挑战建筑中各种挑战boss图标
/archiver_4 存档挑战建筑4: 4存档挑战建筑中各种挑战boss图标


各个类的相关功能和操作

ThiefController: 小偷操作类
1. 通过'A' 键, 攻击金矿操作
2. 丢弃物品操作
3. 范围拾取物品操作
4. 购买商店的物品(商店只有这一个操作与之关联, 放入到小偷操作类中)

LockerController: 储物柜操作类
1. 拾取丢弃点的物品
2. 购买技能书晕锤
3. 丢弃物品栏物品到指定丢弃点

DifficultyController: 难度操作类
1. 选择难度
2. 选择模式
3. 选择挑战

StrengthenAttributeController: 强化属性类
1. 每隔1分钟执行一次属性强化, 判断时间是否超过15分钟
2. 强化各项属性

BossChallengeController: boss 挑战操作类
1. 开启自动挑战金币怪
2. 到达15分钟后, 开启最终boss 挑战
3. 到达16分钟后, (15分钟 + 1分钟等待), 每隔10判断游戏胜利 按钮是否存在


ModifierController: 修改器操作类
1. 找到'物品信息' 信息栏, 返回坐标
2. 通过计算定位物品坐标
3. 修改物品值

ArchiverChallengeController: 存档boss 挑战操作类
1. 点击不同的存档挑战建筑
2. 点击存档建筑中的存档挑战
3. 等待3分钟 左右结束游戏


CommonController: 公共操作类
1. 查找图片并且返回图片坐标
2. 查找图片并且点击图片
3. 查找图片是否存在
4. 移动鼠标到指定位置
5. 键入值
6. 组合键
7. 编号操作

GameFlowController: 游戏流程控制类

ThiefGameStarter: 启动类


窗口切换可以用 com.marks.tools.kkplatform.WindowSwitcherUtils
准备房间可以用 com.marks.tools.kkplatform.PrepareRoom
