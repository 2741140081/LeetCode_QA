update: 2026-03-17
1. 将当前两个自动化游戏的单体包, 修改为一个spring boot 项目, 需要创建一个新的spring boot 项目, 项目名称: kk平台游戏自动化(kkPlatformGameAuto)
2. 启动方式通过访问前端 game.html 页面, 下拉框中选择游戏名, 选择难度, 选择执行次数
3. 将一些公共操作方法进行提取, 例如准备房间的操作, 窗口切换操作, 修改器操作, 图片识别操作, 点击操作, 移动操作, cmd命令操作, 这些操作都是公共操作, 不同的游戏都可以使用这些操作
4. 这些公共操作可以通过spring 的依赖注入, 实现解耦
5. 由于当前路径的硬编码, 需要添加配置文件, 和读取配置文件的类
6. 不同游戏的所需的图片是不同的, 对于这些不同游戏所含的图片路径, 需要分别配置不同游戏配置文件存储这些图片路径
