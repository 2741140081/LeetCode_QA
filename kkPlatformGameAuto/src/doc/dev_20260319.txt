ThiefGameRequest: 请求参数的实体类
1. 游戏名称: String 类型
2. 游戏难度: int 类型
3. 游戏模式: int 类型(前端是一个单选框, 选项有[模式0, 模式1, ..., 模式4])
4. 开启的挑战类型: List<Integer>, 一个多选框[0 ~ 6], 其中0标识不开启任何挑战
5. 选择要挑战的存档建筑: List<Integer> 多选[0~6], 0表示不挑战任何存档建筑
6. 执行游戏次数: int,
7. 运行完成后是否关闭计算机: boolean/int, 具体看前端

2. 游戏模式

5. 将小偷的行为抽象成一个服务, HeroOperationService(英雄操作服务)
    1. 'A' 键攻击行为: 传入一个 Point targetPoint; 攻击目标地点, 这个功能与释放技能重复, (删除该功能)
    2. 释放技能行为: 有两种释放技能的方式
    [通过图片识别技能图片, 然后点击技能图标, 移动到目标点, 释放技能]: 需要参数(技能图片完整路径 imagePath, 释放地点 Point)
    [通过游戏系统内已设置的键盘快捷键释放技能, 点击'快捷键', 移动到目标点, 释放技能]: 需要参数(int keyCode, 释放地点 Point)
    3. 移交物品栏物品到目标点(Point):
    移交一件物品到目标点: 需要参数(物品栏物品的图片完整路径, 目标点)
    移交物品栏所有相同的物品到目标点: 可以直接调用(移交一件物品到目标点), 然后判断物品栏是否还有该物品, 重复执行和判断
    移交物品栏所有物品(物品可以不相同)到目标点: 通过物品栏上方的 logo 图片, 定位该图片位置, 得到物品栏所有物品Point:
    需要参数(目标点), logo 图片是从配置文件中获取。

6. 需要添加新的配置文件, 用来设置小偷游戏的相关参数
    1. 图片存放目录
    2. 小偷编号: 1; 储物柜编号: 2; 最终boss挑战建筑编号: 3; 属性强化建筑编号: 4;
    3. 存档建筑物编号: [1~6], 通过查找小偷游戏的图片目录, 找到 "archiver_" + number(编号数字) 案找图片建筑并且编号
    4. 装备物品名称: w290, w291, w293, IOOC 等物品图片
    5. 小偷游戏中需要修改物品: 需要一个数组或者 List 结构存储
    第一次修改物品: {'w291'}
    第二次修改物品: {'w291', 'w293', 'w293', 'w290', 'w290'}
    默认修改物品: {'w293', 'w293', 'w293', 'w293', 'w293', 'w293'}
